# 什么是 01 背包问题？

​ 01 背包问题是经典的动态规划问题，他也是一种模板，很多动态规划的题目都是基于 01 背包这个模板来创作出来的。

# 问题描述：

​ 给定一组物品，每个物品有两个属性：重量 (weight) 和 价值 (value)。然后再给定一个固定容量的背包。要求你从这样物品中选择一些物品放入背包 (每个物品只能被选择一次)，使得在不超过背包容量的前提下，选出的物品的总价值达到最大。

​ 为什么叫 01 背包呢？原因是每个物品只能被选择一次。

# 准备工作

​ 首先既然给定了一组物品，那么我们可以利用数组，按照物品对应的序号，将物品的价值和物品的重量分别放入一个一维数组中，以便之后的操作。

​ 数组 value [N] 存储了物品编号为 N 的物品的价值。

​ 数组 weight [N] 存储了物品编号为 N 的物品的重量。

​ 一个变量或者常量 capacity 表示背包的容量。

​ 一个二维 dp 数组 dp [i] [j]，当前这个二维 dp 数组还没有初始化，但是它的含义是在选择了前 i 个物品并且背包容量为 j 时的最大总价值。其中，i 表示物品的索引， j 表示当前的背包容量。

# 初始化

​ 首先我们要先确定一下物品数量，背包容量。假设物品数量为 N，背包容量为 capacity，则我们定义 dp 数组时我们要这样定义：dp [N+1] [capacity+1]; 这样我们就限制了最大的背包容量（N+1 和 capacity+1 中的 + 1 是为了防止越界）然后我们就可以着手初始化 dp 数组了。

​ 动态规划是根据前面的状态推出后面的状态，我们先假设 dp [0] [0]。dp [0] [0] 代表前 0 个物品并且背包容量为 0 时的最大总价值，但是由于是前 0 个物品且背包容量为 0，所以无法容纳任何东西（即使有重量为 0 的物品，但是前面的前 0 个物品也使得无法在这里存储这个重量为 0 的物品，正常人都不会让某个物品的重量为 0 的）

​ 所以二维 dp 的外层只要是 0 的都可以初始化为 0。例如 dp [0] [0] ~ dp [0] [capacity+1] 都可以初始化为 0。

​ 并且二维 dp 的内层只要是 0 的都可以初始化为 0。例如 dp [0] [0] ~ dp [N+1] [0] 都可以初始化为 0。

​ 接下来就轮到内层和外层都不为 0 的情况了，由于背包一开始并不存放东西，所以我们可以放心的都初始化为 0。

​ 这样我们就得到一个结论，将 dp 数组全部初始化为 0。

​

# 状态转移方程

​ 回忆一下：我们通常使用一个二维的 DP 数组 dp [i] [j] 来表示在选择了前 i 个物品并且背包容量为 j 时的最大总价值。其中，i 表示物品的索引， j 表示当前的背包容量。

  • 当 i = 0 或 j = 0 时，dp [i] [j]=0，表示背包容量为 0 或没有物品可选时，最优解为 0

  • 当 weight [i] > j 时，即第 i 个物品的重量大于当前背包的容量 j 时，dp [i] [j]=dp [i-1] [j] 表示只考虑前 i-1 个物品时的最优解。

  • 当 weight [i] <= j 时，即第 i 个物品的重量小于等于当前背包容量 j 时，我们需要在选择放入第 i 个物品和不放入第 i 个物品之间进行判断。

    ​ 如果选择放入第 i 个物品，则总价值为 value [i] + dp [i-1] [j-weight [i]]，即当前物品 i 的价 值加上选择前 i-1 个物品，并且使用背包容量减去当前物品重量。

    ​ 如果不选择放入第 i 个物品，则总价值为 dp [i-1] [j]，即值考虑前 i-1 个物品时的最优解。

​ 因此状态转移方程为：dp[i] [j] = max(value[i] + dp[i-1] [j-weight[i]] , dp[i-1] [j]);

# 遍历顺序

​ 回忆一下：01 背包问题，是指每个物品只能选择一次，并且保证被选中的物品重量在不超过背包的容量的前提下达到最大价值。那么如何保证我们每次选取物品时只选取一次呢？

​ 遍历顺序可以保证。01 背包通常使用二维数组进行动态规划求解，代表我们将要使用两层 for 循环来遍历物品和背包。我们只要保证外层 for 遍历物品，内层 for 遍历背包容量 即可保证每个物品只被选择一次。

​ 外层遍历物品：设物品的数量为 N 则从 1 依次遍历到 N 则会依次考虑每个物品。

​ 内层遍历背包容量：设背包容量为 capacity。则从 1 到 capacity，依次考虑每个背包的容量。并且外层遍历的是物品，假设当前遍历到了物品 M，则物品 M 会在 1capacity 这个背包容量区间依次进行判断是否要将这个物品 M 放入容量为 1capacity 的背包。当外层物品 M 该接着往下遍历 M+1 个物品时，M 物品已经根据内层背包容量的循环作出了最优选择，并且填充了相应的 dp 数组。这也是为什么我们可以根据已经计算得到的 dp 数组来推导出是否选择当前物品的最优解。

# 打印 / 输出结果

​ 因为我们要得到前 N 个物品在选择放入背包容量为 capacity 时的最优总价值，所以我们遍历完毕后要输出 dp [N] [capacity] 这个值。

​ 当然如果你想要调试程序或者想要仔细看看 dp 数组的内部，你可以像打印二维数组那样打印出 dp 数组即可。

# 状态压缩

​ 在二维 dp 数组中，当前的 dp [i] [j] 是根据 dp [i-1] [j] 来推出的，因此我们在计算 dp [i] [j] 时只需要涉及上一层和当前层的状态 dp [i-1] [j] 来进行判断。因此我们不必存储整个二维 dp 数组，我们可以维护一个一维数组，这个一维数组也是根据上一层和当前层的状态来推断出来的。

# 一维 DP 含义

​ 一维 dp 数组 dp [j] 来表示容量为 j 的背包的最大价值为 dp [j]。

# 状态转移方程

​ 方程为：dp[j] = max(dp[j] , dp[j - weight[i]] + value[i]);

​ 当我们考虑选择第 i 个物品时，对于每个背包容量 j ，我们有两个选择：

​ 第一个是不选择第 i 个物品，这样的话我们可以让背包保持原样，dp [j] 的值任然等于上一层 i-1 在背包容量为 j 时的最大价值。

​ 第二个是选择第 i 个物品，这样的话我们需要保证物品 i 的重量不超过当前背包的大小，并且物品 i 放入容量为 j 的背包后，它的价值要大于原先的 dp [j]。

​ 通过比较这两种情况下的价值，我们选择一个较大值作为当前层 i 在背包容量为 j 下的最大价值，即 dp [j] 的值。

# 遍历顺序

​ 依旧是先遍历背包，再遍历物品。并且内层循环要倒序遍历。因为倒序遍历可以计算每个状态的最大价值。

​ 如果我们正序遍历的话，会从小容量背包开始慢慢增大。在计算第 i 个物品的价值时，可能会用到第 i-1 个物品时已经更新过的 dp 值。（按道理计算第 i 层应该是结合 i 和 i-1 层的旧值 来计算得到的，但在这里正序可能会直接用到 i-1 的新值，可能会导致重复选择同一个物品，导致违背 01 背包问题的基本原则）

​ 如果我们倒序遍历，那么我们背包容量从高慢慢到底，我们可以保证在计算 dp [j] 时，使用的上一层 i-1 都是旧值（或者默认值 0）。这样就能正确地得出 dp 数组。
